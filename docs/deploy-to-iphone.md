# Deploy to iPhone (Xcode Direct Install)

No App Store or Expo Go required. Installs the app directly to your iPhone via USB.

## Prerequisites

- Mac with Xcode installed
- Apple ID (free — no paid developer account needed)
- iPhone connected via USB
- `bundleIdentifier` set in `app.json` → `expo.ios` ✓ (already done: `com.whattoeat.app`)

## One-Time Xcode Setup

1. Open **Xcode**
2. Go to **Settings → Accounts**
3. Click **+** → **Apple ID** → sign in
4. Close Settings

## Deploy Steps

1. **Connect iPhone via USB** — tap "Trust" on the phone if prompted

2. **From the `what-to-eat` directory, run:**

   ```bash
   npx expo run:ios --device
   ```

   - First run: generates an `ios/` folder and builds from scratch (5–10 min)
   - Subsequent runs: incremental build (faster)
   - When prompted, select your iPhone from the device list

3. **If Xcode shows a signing error:**
   - Open the generated `ios/` folder in Xcode: `open ios/whattoeat.xcworkspace`
   - Select the project in the left panel → **Signing & Capabilities**
   - Set **Team** to your personal Apple ID team
   - Close Xcode and re-run the command

4. **If iPhone shows "Untrusted Developer":**
   - On iPhone: **Settings → VPN & Device Management → your Apple ID → Trust**
   - Re-launch the app

## Subsequent Deploys

Same command each time:

```bash
npx expo run:ios --device
```

No need to re-do the Xcode or trust steps unless you reset your phone or switch machines.

## Notes

- The `ios/` folder generated by `expo run:ios` is gitignored by default — don't commit it
- If you update `app.json` (name, icon, etc.), clean the build: `npx expo run:ios --device --clean`
- Free Apple ID installs expire after **7 days** — just re-run the deploy command to refresh

## Troubleshooting Log (2026-03-28 / 2026-03-29)

### 1) `pod install` fails with `undefined method 'visionos'`

**Error**

- `Invalid react-native-safe-area-context.podspec: undefined method 'visionos'`

**Cause**

- Old CocoaPods version (1.10.0) was being used.

**Fix**

1. Install/upgrade CocoaPods with Homebrew.
2. Ensure the Homebrew `pod` binary is the one used.
3. Reinstall pods.

```bash
brew install cocoapods || brew upgrade cocoapods
brew link --overwrite cocoapods
which pod
pod --version  # should be 1.14+ (used 1.16.2)

cd ios
rm -rf Pods Podfile.lock
pod install --repo-update
```

---

### 2) Expo could not detect iPhone (`No iOS devices available in Simulator.app`)

**Error**

- `Unexpected devicectl JSON version output from devicectl`
- `CommandError: No iOS devices available in Simulator.app`

**Cause**

- Expo device detection was unreliable with current `devicectl` output.

**Fix**

- Verify device visibility with Apple tools and run with explicit UDID.
- If needed, build once from Xcode directly.

```bash
xcrun xctrace list devices
npx expo run:ios --device <YOUR_UDID>
```

Example UDID used:

- `00008120-000C69A60E32201E`

---

### 3) Code signing errors

#### A) `No code signing certificates are available to use`

**Fix**

- In Xcode: **Settings → Accounts → Manage Certificates → + Apple Development**
- In target Signing: enable **Automatically manage signing**, select Team.

#### B) App ID not available (`com.whattoeat.app`)

**Error**

- `The app identifier "com.whattoeat.app" cannot be registered...`

**Fix**

- Use a unique bundle identifier (reverse-DNS).
- Keep identifier aligned in both Expo config and Xcode target.

Example:

- `com.marianela.whattoeat.dev`

---

### 4) Provisioning profile error

**Error**

- `No profiles for 'com.whattoeat.app' were found`
- `Automatic signing is disabled`

**Fix**

- In Xcode target **Signing & Capabilities**:
  - Enable **Automatically manage signing**
  - Select Team
  - Use unique bundle ID

---

### 5) ReactCodegen missing generated files

**Error examples**

- `Build input file cannot be found ... rnscreens-generated.mm`
- `... safeareacontext-generated.mm`
- `... rnasyncstorage-generated.mm`

**Cause**

- Stale iOS artifacts and/or package version mismatch after manual downgrades.

**Fix**

- Restore Expo-compatible package versions, clean generated artifacts, reinstall pods.

```bash
npx expo install react-native-safe-area-context react-native-screens @react-native-async-storage/async-storage
rm -rf ios/build ios/Pods ios/Podfile.lock
rm -rf ~/Library/Developer/Xcode/DerivedData/*
cd ios && pod install --repo-update && cd ..
npx expo run:ios --device 00008120-000C69A60E32201E
```

---

### 6) App installs but shows runtime error on launch

**Cause**

- Dev build opened without Metro/dev server attached.

**Fix**

```bash
npx expo start --dev-client --clear
# if networking issues:
npx expo start --dev-client --tunnel --clear
```

---

### 7) App deployed but not visible on Home Screen

**Checks**

- Search app by name (Spotlight / App Library).
- iPhone setting: **Home Screen & App Library → Newly Downloaded Apps → Add to Home Screen**.
- Remove stale old app install and rebuild if needed.
